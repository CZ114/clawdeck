关键决策对照

/ at-a-glance — v0 取向 vs 留待 Stage 2
维度 v0 取向 舍弃 / 留待 Stage 2
生成时机 每次首次开 bubble + 22:00 自动 + dashboard ↻ 手动；首次有同意确认 session 结束实时生成成本不可控，留 Stage 2 看必要性
模型来源 claude -p 子进程，复用本地 Claude Code 已认证的会话 不直接调 Anthropic API，无需让用户配 key
题型 选择题（默认） + 填空题（命令 / 公式 / 常量） 简答题需二次调模型，token 翻倍；Stage 2 再加
复习节奏 每张卡有 attempts 历史；错题自动入"明天"队列 完整 Anki 式 SRS（间隔重复）等用一两周再说
UI 入口 Dashboard "Today's review" section + 复用 approval 模式 360 ×~330 卡片 Done attention 之后顺势弹卡片留 Stage 2
隐私 送给 claude -p 前先脱敏：去 .env 路径段落、token-like 字符串、绝对路径用户名 明确把 review 数据流标为 docs/security.md 的 opt-in 例外
总开关 ~/.claude-companion/cards-disabled 哨兵 + bubble 设置 toggle 跟现有 disabled 哨兵同模式，第一原则是用户能轻松停掉
